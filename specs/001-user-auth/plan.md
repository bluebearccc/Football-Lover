# Implementation Plan: User Authentication

**Branch**: `001-user-auth` | **Date**: 2026-06-21 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/001-user-auth/spec.md`

## Summary

Implement UC01 (Register), UC02 (Login/Logout), and UC15 (Password Reset) for the Football-Lover platform. The backend auth module already exists with core flows (register, login, logout, forgot/reset password). This plan addresses **9 gaps** between the current implementation and the refined spec: changing post-registration to redirect-to-login (no auto-login), adding rate limiting on login/reset endpoints, adding case-insensitive email handling, invalidating old reset tokens, adding a password safety checklist component, creating an admin seed script, adding Terms of Use notice, wiring SMTP in the mailer, and aligning frontend UI with Stitch mockups.

## Technical Context

**Language/Version**: TypeScript (strict mode, both backend and frontend)

**Primary Dependencies**:
- Backend: Express, Prisma 6.x, Zod, bcryptjs, jsonwebtoken, nodemailer (to add)
- Frontend: Next.js 14.2.x (App Router), TailwindCSS, Material Symbols Outlined icons

**Storage**: PostgreSQL via Prisma. User + PasswordResetToken models already in schema.

**Testing**: Build check (`npm run build`) + lint (`npm run lint`) in both packages. Manual validation per `quickstart.md`.

**Target Platform**: Web browser (responsive, Vietnamese locale)

**Project Type**: Web application (backend API + frontend SPA)

**Performance Goals**: Login < 1s (P95), Register < 1s (P95), Password reset < 5s (P95)

**Constraints**: ~100 users, 5–20 concurrent. Prisma v6 pinned, Next.js 14.2.x pinned.

**Scale/Scope**: 4 screens (login, register, forgot-password, reset-password), 6 API endpoints, 2 entities
